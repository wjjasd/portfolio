## 3. 사이트 구성 (정보 설계)

### 3.1 페이지 구조

```
포트폴리오 웹사이트
├── Hero Section       ← 첫인상 (이름 + 한 줄 소개 + CTA)
├── About              ← 나는 누구인가 (스토리 + 사진)
├── Skills             ← 기술 스택 시각화
├── Projects           ← 핵심 프로젝트 3~5개 (상세 페이지 포함)
├── Experience         ← 경력 타임라인
├── Blog/Articles      ← 기술 블로그 연동 (Tistory RSS: dev-yangkj.tistory.com)
├── Contact            ← 문의 채널 (이메일, LinkedIn, GitHub)
└── Resume             ← PDF 이력서 다운로드 버튼
```

### 3.2 핵심 콘텐츠 전략

**Projects 섹션이 가장 중요** — 채용 담당자는 평균 6초 내에 판단

각 프로젝트 카드에 반드시 포함할 정보:
- 한 줄 요약 (What + Why)
- 사용 기술 스택 태그
- **정량적 성과** (예: "DAU 5,000명", "API 응답 50% 단축")
- GitHub 링크 + 라이브 데모 링크
- 나의 역할 (팀 프로젝트인 경우)

## 5. 기술 스택 선정

### 5.1 권장 스택

| 영역 | 기술 | 선정 이유 |
|------|------|----------|
| 프레임워크 | **Next.js 14 (App Router)** | SSR/SSG 혼용 → SEO 최적화, Vercel 무료 배포 |
| 언어 | **TypeScript** | 미드레벨 역량 어필, 유지보수성 |
| 스타일링 | **Tailwind CSS** | 빠른 개발 속도, 반응형 편의성 |
| 애니메이션 | **Framer Motion** | 부드러운 UX, 간단한 API |
| 디자인 (1차 시안) | **Google Stitch** | AI 프롬프트로 빠른 시안 생성, Tailwind/HTML 코드 직접 추출 가능, 현재 무료 |
| 디자인 (상세 편집) | **Figma** (무료 플랜) | 픽셀 단위 조정, Stitch에서 Export 후 마무리 작업 |
| 배포 | **Vercel** | Next.js 최적화, 무료 티어, CI/CD 자동화 |
| 문의 폼 | **EmailJS** 또는 **Resend** | 서버 없이 이메일 발송 |
| 블로그 연동 | **Tistory RSS** (`/rss`) | 별도 API 없이 바로 파싱 가능, 무료 |
| 분석 | **Google Analytics 4** | 방문자 추적 |

### 5.2 디자인 워크플로우 (Stitch + Figma 병행)

```
[1단계] Google Stitch — 시안 생성 (20~30분)
  텍스트 프롬프트 입력
  → Hero, Projects, About 섹션 시안 자동 생성
  → 마음에 드는 방향 선택
  → Tailwind CSS 코드 직접 Export → Next.js에 바로 적용 가능

[2단계] Figma — 디테일 다듬기 (필요 시)
  Stitch에서 Figma 포맷으로 Export
  → 컴포넌트 정렬, 간격, 색상 정밀 조정
  → 개발자 핸드오프용 스펙 확인

[3단계] 개발 — 코드 적용
  Stitch Export 코드 또는 Figma 스펙 기반으로 구현
```

**Stitch 프롬프트 예시 (포트폴리오용):**
> "A dark-themed developer portfolio hero section with name, one-line bio, and two CTA buttons. Clean, minimalist design using Tailwind CSS. Accent color: indigo."

- 과도한 화려함 ❌ → 명확성·가독성 중심 ✅
- 어두운 배경(다크 테마)이 개발자 포트폴리오에서 선호도 높음
- 컬러 포인트 1개 (예: 브랜드 색상)로 일관성 유지
